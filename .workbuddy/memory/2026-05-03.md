## 小程序全部 JS 文件 ES5 改造

**原因**：`@babel/runtime` 依赖缺失，ES6 语法（解构赋值、箭头函数、展开运算符、async/await、模板字符串）在小程序基础库中无法运行。

**改造范围**（10 个 JS 文件 + 1 个公共模块）：
- `utils/dbInit.js` - ES5 改写
- `app.js` - 改为调用云函数初始化
- `pages/onboarding/onboarding.js` - 完全自包含，不依赖 dbInit
- `pages/toolbox/toolbox.js` - ES5 + Promise 链
- `pages/toolManager/toolManager.js` - ES5 + Promise 链
- `pages/submissions/submissions.js` - ES5 + Promise 链
- `pages/reviews/reviews.js` - ES5 + Promise 链
- `pages/conferences/conferences.js` - ES5 + Promise 链
- `pages/archive/archive.js` - ES5 + Promise 链
- `pages/home/home.js` - ES5 + Promise 链（修复 goToToolbox 重复定义 bug）
- `pages/calendar/calendar.js` - ES5 + Promise 链

**规则**：
- 不用 `const`/`let` → 全部用 `var`
- 不用箭头函数 `=>` → 用 `function()`
- 不用解构 `const { a, b } = obj` → 用 `var dbInit = require(); var a = dbInit.a;`
- 不用展开运算符 `...obj` → 用 `Object.assign()` 或手动赋值
- 不用模板字符串 `` `${}` `` → 用字符串拼接 `+`
- 不用 `async/await` → 用 Promise 链 `.then().catch()`
- 不用 `.includes()` → 用 `.indexOf() !== -1`
- `notDeleted` 查询改为 `deleteTime: null`（不用 command.eq(null)）
- `dbInit.js` 用 `module.exports = { key: value }` 导出

## 云函数拆分为两个

**拆分原因**：初始化逻辑和业务方法混在一起，不好维护。

**academicTools**（数据库初始化）：
- `initDB`：创建集合 + upsert 写入 tools 表（已有数据跳过）
- `resetDB`：清空所有集合并重新初始化
- 调用方式：`wx.cloud.callFunction({ name: 'academicTools', data: { type: 'initDB' } })`

**academicAPI**（业务接口，新建）：
- `getUserId`：获取当前用户 OpenID
- `getUserConfig`：获取用户配置
- `saveUserConfig`：保存/更新用户配置（_openid 唯一）
- `getUserTools`：获取用户工具配置（按 _openid 过滤）
- `toggleUserTool`：切换工具开关（_openid + toolId 唯一）
- `saveUserTools`：批量保存工具配置（onboarding 时用）
- 调用方式：`wx.cloud.callFunction({ name: 'academicAPI', data: { action: 'getUserTools' } })`

**客户端调用路径更新**：
- `app.js` → `academicTools` (initDB)
- `onboarding.js` → `academicAPI` (saveUserConfig, saveUserTools)
- `toolbox.js` → `academicAPI` (getUserTools)
- `toolManager.js` → `academicAPI` (getUserTools, toggleUserTool)
