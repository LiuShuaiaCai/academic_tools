## 投稿管理 - 高级筛选选项修复

### 问题
高级筛选选项（状态/身份/期刊/优先级）受到快速筛选按钮（全部/未完成/急需处理/特别紧急）的影响。
原因：`processItems` 里用不含 quickFilter 的数据正确构建了选项，但 `applyFilter()` 在 setData 回调里又用 `this.data.list`（含 quickFilter）重新覆盖了选项。

### 修复方案
1. **`applyFilter` 不再重建高级筛选选项**，只更新 `filteredList`
2. **高级筛选选项只在 `processItems` 里构建**（非加载更多时），优先使用不含 quickFilter 的 `optionsRes.data`

### 技术细节
- `loadList` 使用双查询并行：`listQuery`（含 quickFilter）+ `optionsQuery`（不含 quickFilter）
- `processItems` 中：`optionsRes && optionsRes.data` 时用无 quickFilter 数据构建选项，否则用当前 list
- `applyFilter` 现在只有一行：`this.setData({ filteredList: baseList });`

## aiService 适配分组名/模型名分离

### 后台实际配置
| 分组名（createModel 用） | 实际模型名（model 字段） |
|------------------------|----------------------|
| `deepseek` | `deepseek-v3-0324` |

| `hunyuan-exp` | `hunyuan-turbos-latest` |

### 核心改动（aiService/index.js）
1. `AI_MODELS` 的 key = 分组名，每个配置加 `model` 字段存实际模型名：
   ```javascript

     model: 'deepseek-v3-0324',  // 实际模型名
     name: 'DeepSeek V3',
     ...
   }
   ```
2. `callAI(groupName, paperText)`：
   - `ai.createModel(groupName)` — 用分组名
   - `model: config.model` — 用实际模型名
3. `getProviders()` 返回的 `model` 字段改为 `p.model`（实际模型名）
4. `DEFAULT_MODEL = 'deepseek'`（分组名）

- `selectedAiProvider` 默认值改为分组名 `'deepseek'`
- `loadAiProviders` 的 fallback 改为 `'deepseek'`
- 前端传的 `provider` 参数 = 分组名


## AI 审稿最终架构：云函数提取文本 + 小程序端调 AI

### 根本原因
`cloud.AI.createModel()` 在云函数端**不稳定**——有时能用有时报 `Cannot read properties of undefined (reading 'AI')`。
官方文档明确 `wx.cloud.extend.AI` 是**小程序端 API**（基础库 ≥ 3.7.1），云函数端没有官方支持。

### 最终架构
- **云函数 `aiService`**：只保留 `extractText`（提取文本）+ `getProviders`（返回模型列表）

## generateText 参数格式 Bug 修复

### 问题
调用 `model.generateText()` 报 400 错误：`AI_MODEL_PARAM_REQUIRED` — Request body missing required parameter: model。
两个模型（deepseek / hunyuan-exp）都报同样的错。

### 原因
`generateText` 参数直接传对象 `{ model, messages }`，**不需要**包在 `data: {}` 里。
`streamText` 才需要 `{ data: { model, messages } }` 格式。
之前错误地给 `generateText` 也包了 `data`，导致实际请求体变成 `{ data: { model: ... } }`，后端找不到 `model` 字段。

### 修复
```javascript
// 修复前（错误）
model.generateText({ data: { model: provider.model, messages: [...] } })

// 修复后（正确）
model.generateText({ model: provider.model, messages: [...] })
```
