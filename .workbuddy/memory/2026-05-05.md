# 2026-05-05 工作记录

## 投稿管理 - 高级筛选选项修复

### 问题
高级筛选选项（状态/身份/期刊/优先级）受到快速筛选按钮（全部/未完成/急需处理/特别紧急）的影响。
原因：`processItems` 里用不含 quickFilter 的数据正确构建了选项，但 `applyFilter()` 在 setData 回调里又用 `this.data.list`（含 quickFilter）重新覆盖了选项。

### 修复方案
1. **`applyFilter` 不再重建高级筛选选项**，只更新 `filteredList`
2. **高级筛选选项只在 `processItems` 里构建**（非加载更多时），优先使用不含 quickFilter 的 `optionsRes.data`

### 技术细节
- `loadList` 使用双查询并行：`listQuery`（含 quickFilter）+ `optionsQuery`（不含 quickFilter）
- `processItems` 中：`optionsRes && optionsRes.data` 时用无 quickFilter 数据构建选项，否则用当前 list
- `applyFilter` 现在只有一行：`this.setData({ filteredList: baseList });`

## 审稿表单字段补全

### 新增字段
- **邀请时间** (invitedDate) — 日期选择器
- **轮次** (round) — 下拉选择 R1~R5+（ROUND_OPTIONS）
- **审稿决定** (decision) — 扩展为 7 个选项：待定/同行/接收/大修/小修/拒绝/其他
- **审稿系统链接** (systemUrl) — 文本输入
- **审稿系统账号** (systemAccount) — 可选，文本输入
- **审稿系统密码** (systemPassword) — 可选，密码输入
- **关联审稿** (relatedReviewId) — 下拉选择（同一稿件多轮审稿）
- **处理时间线** (timeline) — 动态数组，事件类型：收到邀请/接受审稿/开始审稿/提交审稿/修回通知/审稿完成/拒绝邀请/其他

- `form.js` — 新增 form 字段、loadRelatedReviews、onRoundChange/onDecisionChange/onRelatedReviewChange/addTimelineItem 等方法
- `form.wxml` — 表单补全所有新字段 UI，决定弹窗改为两行 7 个按钮
