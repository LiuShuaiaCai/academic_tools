## 2026-05-04 晚间更新

### 投稿列表卡片截止日期显示逻辑优化
- 文件：`utils/submissions-format.js` 第59行 `overdueLabel`
- 新逻辑：
  - **已过期**（overdue < 0）：显示「超过X天」
  - **剩余0-3天**（0 ≤ overdue ≤ 3）：显示「剩余X天」
  - **剩余>3天**（overdue > 3）：显示到期日期（格式：YYYY-MM-DD）
- 原逻辑：只区分已超期/剩余天数，无日期展示

### 全量统计改造（方案A）
- **云函数**：`academicAPI/index.js` 新增 `submissionStats` action，遍历全部未删除投稿，统计 total/incomplete/near/urgent 四个计数
- **小程序端**：`submissions.js` 新增 `loadStats()` 方法，首次加载时调用云函数获取全量统计数字
- **去掉了** `processItems` 中的本地前端统计逻辑
- 统计数字现在基于数据库全量数据，不受分页20条限制
- 注意：云函数修改后需重新上传部署才能生效

### 修复语法错误
- `submissions.js` 第139行存在多余的 `}`，导致 `loadStats` 方法体内出现双重 `},` 结尾
- 多余的 `}` 提前关闭了 `Page({})` 对象，使 `onReachBottom` 等方法位于对象外部，触发语法错误
- 修复方式：删除第139行的多余 `}`

### 修复 submissions.wxml 结构错误
- `Component is not found in path "wx://not-found"` 的根因：wxml 中 `.sub-list` 的 `</view>` 位置错误（第155行多了一个额外的 `</view>`），导致 `load-more`、`fab`、`submission-form` 组件被推出正确层级，组件路径解析失败降级为 `wx://not-found`
- 修复方式：删除多余的 `</view>`，让 `.sub-list` 正确在 card-footer 的 `</view>` 后闭合

### 修复 near/urgent 天数逻辑
- 原问题：云函数 `near` 条件为 `days >= 1 && days <= 3`，导致 days=1 同时进入 near 和 urgent 被双计；前端 `near` 为 `days >= 0 && days <= 3`，把今日到期也算进紧急处理，语义错误
- 修复后统一标准：
  - **特别紧急（urgent）**：days 0~1（今天或明天到期）
  - **紧急处理（near）**：days 2~3（后天或大后天到期）
- 同步修改了云函数 `academicAPI/index.js` 和前端 `submissions.js` 的 applyFilter 逻辑
- **注意**：云函数修改后需重新上传部署

### 再次修正 days 计算（统一按日期差）
- 问题：云函数上传后 urgent 还是 0，发现 `Math.ceil` 受当天时间影响不稳定
- 修复：改用 `Math.round`，并统一前后端按"日期字符串差"计算（忽略时间部分）：
  - 取 `deadline` 前10位（YYYY-MM-DD），补 `T00:00:00` 生成 Date
  - 取今天日期（00:00:00）生成 Date
  - `days = Math.round((dlDate - todayDate) / 86400000)`
- 云函数 `submissionStats` 加了 `console.log` 调试日志，上传后可在云函数日志查看每条投稿的 title/deadline/days/completed 值
- 前端 `submissions.js` applyFilter 同步修改，保持前后端一致
- 定义：urgent=0~1天，near=2~3天

### 修复 completed 自动计算 bug（根因）
- 现象：云函数返回 `incomplete:0, urgent:0`，2条投稿全部 `completed:true`
- 根因：`form.js` saveForm 中自动计算 completed 逻辑错误——只要时间线最新事件日期 >= deadline 就标为完成，太激进
- 修复：改为只有时间线包含终态事件（接收/发表/出版/online/accepted/published）才自动标 completed=true；编辑时保留原值不覆盖
- 新增云函数 `fixCompleted` action，批量修复已有错误数据
- 调用方式：`wx.cloud.callFunction({ name:'academicAPI', data:{ action:'fixCompleted' } })`
