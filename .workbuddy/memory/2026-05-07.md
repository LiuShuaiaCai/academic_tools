## archive.js 文件缓存机制
- 新增 `_getFilePath(file, callback)`：优先查 `fileCache`（内存），命中则直接用；未命中则 `downloadFile` → `saveFile` 持久化到本地，再缓存
- `previewFile`（文档）和 `shareFile` 全部改为走 `_getFilePath`，同一文件只需下载一次
- 图片/HTML/MD/XML 不走缓存（图片直接用 fileID，其他走临时预览页）

## archive.js 分享功能
- `wx.shareFileMessage` 有平台限制：必须在用户手势同步上下文中直接调用，异步回调里调用会失败并退化为小程序分享
- 分享方案（两级兜底）：
  - 第1级：有本地缓存 → `shareFileMessage` 直接分享文件（快）
  - 第2级：无缓存或失败 → 分享小程序卡片（`pendingShare` → `onShareAppMessage` → 卡片带 `fileId` 参数）
  - 第3级：好友点卡片 → `onLoad` 接收 `fileId` → `_autoDownloadSharedFile` 自动下载并打开
- `pendingShare` 字段：存储待分享文件信息，供 `onShareAppMessage` 读取生成卡片
- `onLoad` 支持 `?fileId=` 参数：通过卡片打开时自动触发下载
