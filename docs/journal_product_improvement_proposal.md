# 期刊查询功能产品改进方案

> 从用户视角重新定义期刊查询的核心价值，明确改进方向和数据策略
> 撰写时间：2026-06-27

---

## 一、用户分析

### 1.1 核心用户画像

| 用户类型 | 占比 | 核心目标 | 使用频率 |
|---------|------|---------|---------|
| **投稿选刊的研究生/博士生** | 60% | 找到合适的期刊投稿 | 高频（投稿前密集使用） |
| **查期刊信息的科研人员** | 30% | 快速查询期刊指标 | 中频（写论文/填表时） |
| **科研管理人员/图书馆员** | 10% | 批量查询/管理 | 低频 |

### 1.2 选刊场景的用户决策路径

```
我有论文要投稿
    ↓
我的研究方向是什么？（学科分类）
    ↓
这个方向的期刊有哪些？（列表+筛选）
    ↓
这个期刊水平如何？（影响因子/分区←核心决策依据）
    ↓
录用难度如何？（录用率/审稿周期）
    ↓
发表费用多少？（APC/是否OA）
    ↓
最终决定投稿
```

**关键发现**：影响因子和分区是选刊的**第一决策依据**，没有这个数据，其他所有功能价值都大打折扣。

## 二、用户痛点分析

### 2.1 现有产品的核心问题

| 问题 | 用户感受 | 严重程度 |
|------|---------|---------|
| 没有影响因子和分区 | "这期刊到底什么水平？不知道" | 🔴 致命 |
| 学科分类展示的是 Topic（如"Large Language Models"） | "我要找医学期刊，不是找 LLM 期刊" | 🔴 致命 |
| 核心指标卡片显示"总发文量、总被引量、h指数" | "这些数对我选刊有什么用？" | 🟡 严重 |
| "健康评分"概念模糊 | "期刊健康是什么？没听过这个概念" | 🟡 严重 |
| 无法按学科分类筛选 | "我想看医学的期刊，只能一页页翻" | 🟡 严重 |
| 详情页信息杂乱，重点不突出 | "找关键信息要翻半天" | 🟠 中等 |

### 2.2 用户最想看的数据（按优先级）

**第一优先级（必须有）**：
- ✅ 影响因子（JIF）
- ✅ JCR分区（Q1/Q2/Q3/Q4）
- ✅ 中科院分区（大类+小类，1区/2区/3区/4区）
- ✅ 学科分类（大类如"医学"，小类如"肿瘤学"）

**第二优先级（很有用）**：
- ✅ 录用率（Acceptance Rate）
- ✅ 审稿周期（从投稿到录用的平均时间）
- ✅ 自引率（判断期刊是否安全）
- ✅ 年发文量趋势（判断期刊是否水刊）

**第三优先级（有用但不紧急）**：
- 出版社
- ISSN
- 官网链接
- OA/APC费用
- DOAJ/MEDLINE收录状态

## 三、竞品分析：知研学术

> 用户提供的参考设计，代表了用户对"好用"的认知

### 3.1 知研学术的核心设计亮点

**列表页**：
- 显示期刊名 + 影响因子 + JCR分区（Q1-Q4用不同颜色标识）
- 显示中科院分区（如"医学1区"）
- 显示学科标签（如"医学"、"生物学"）
- 卡片简洁，一眼能看到核心决策信息

**详情页**：
- 顶部大字号显示影响因子和分区
- 清晰的学科分类展示（大类+小类）
- 趋势图（影响因子年度趋势）
- 自引率等关键指标

### 3.2 我们可以借鉴的点

1. **列表卡片必须显示 IF + 分区**（这是用户第一眼要看的）
2. **分区用颜色区分**（Q1绿色、Q2蓝色、Q3黄色、Q4灰色）
3. **学科标签要简洁**（大类即可，如"医学"，不要显示细粒度 Topic）
4. **详情页核心指标置顶**（IF + 分区 + 录用率，大字显示）

## 四、现有数据盘点与缺口

### 4.1 当前已有数据（OpenAlex 免费）

| 数据项 | 字段 | 用户价值 | 说明 |
|--------|------|---------|------|
| 总发文量 | `works_count_latest` | 低 | 用户不关心绝对数量 |
| 总被引量 | `cited_by_count_latest` | 低 | 用户不关心绝对数量 |
| h指数 | `h_index_latest` | 低 | 用户不熟悉这个概念 |
| i10指数 | `i10_index_latest` | 低 | 用户不熟悉这个概念 |
| 2年平均被引 | `two_year_mean_citedness_latest` | 中 | 可近似替代 CiteScore |
| Topics | `topics` | 低 | 太细，不是用户认知的学科 |
| OA状态 | `is_open_access` | 中 | 选刊时有用 |
| APC费用 | `apc_amount` | 中 | 选刊时有用 |

### 4.2 缺失的关键数据

| 数据项 | 重要性 | 获取方案 |
|--------|---------|---------|
| **影响因子（JIF）** | 🔴 必须有 | 方案A：爬取公开数据；方案B：用户认领填写；方案C：手动录入 Top 期刊 |
| **JCR分区** | 🔴 必须有 | 随 IF 一起获取（JCR 报告中有 Quartile） |
| **中科院分区** | 🔴 必须有 | 需单独获取（中科院每年发布） |
| **学科大类/小类** | 🔴 必须有 | 从 OpenAlex `topics[].field` 和 `subfield` 提取 |
| **录用率** | 🟡 很有用 | 方案A：用户贡献；方案B：爬取官网/LetPub |
| **审稿周期** | 🟡 很有用 | 方案A：用户贡献；方案B：爬取官网 |
| **自引率** | 🟡 很有用 | 可从 OpenAlex works 数据计算（需额外 API 调用） |
| **CiteScore** | 🟢 有用 | 可用 `cited_by_count / works_count` 近似计算 |

### 4.3 数据获取策略（分阶段）

**第一阶段（立即执行）**：手动录入 Top 200 期刊的 IF/分区数据
- 选择用户搜索量最高的 200 本期刊
- 从 JCR 官网/知研学术/LetPub 等公开渠道获取
- 录入到 `journal_metrics_manual` 表

**第二阶段（短期）**：开发用户贡献功能
- 用户可以对期刊"认领"并填写指标数据
- 需要审核机制（防止恶意填写）
- 贡献者获得积分/徽章

**第三阶段（中期）**：爬取公开数据源
- 爬取 JCR 公开数据（需注意版权）
- 对接 LetPub API（如果有）
- 爬取期刊官网的 Editorial Info

**第四阶段（长期）**：接入付费数据源
- Clarivate JCR API（付费）
- 中科院分区数据（付费）

## 五、产品改进方案

### 5.1 改进原则

1. **核心指标置顶**：IF、分区、学科分类必须一眼能看到
2. **去除无效信息**：去掉"健康评分"、i10指数等对用户无价值的内容
3. **参考但不复制**：借鉴知研学术的设计思路，但适配小程序的交互习惯
4. **数据渐进式填充**：先手动录入 Top 期刊，再逐步扩大覆盖范围

### 5.2 列表页改进

**当前问题**：
- 显示"年发文、被引、h指数、2年均引"——用户不关心
- 没有学科标签
- 没有分区/IF标识

**改进方案**：

```
┌─────────────────────────────────────┐
│ [Q1] Nature                        │  ← 分区标识（颜色区分）
│ 医学 · 肿瘤学                      │  ← 学科分类（大类·小类）
│ IF: 42.7 | 中科院1区               │  ← 核心指标
│ Nature Publishing Group · ISSN 0028-0836 │
│ OA · APC: £9,990                  │
└─────────────────────────────────────┘
```

**关键改动**：
1. 卡片顶部加分区标识（Q1/Q2/Q3/Q4，不同颜色）
2. 标题下方显示学科分类（大类·小类）
3. 显示影响因子（有数据的话）或"暂未收录"
4. 去掉"年发文、被引、h指数"指标
5. 筛选面板加"学科分类"入口

### 5.3 详情页改进

**当前问题**：
- "核心指标卡片"显示的数据用户不关心
- "健康评分"概念奇怪
- 学科主题显示的是 Topic（太细）

**改进方案**：

```
┌─────────────────────────────────────┐
│ Nature                             │
│ Nature Publishing Group · 英国      │
├─────────────────────────────────────┤
│                                   │
│  影响因子  42.7     JCR Q1         │  ← 核心指标大字显示
│  中科院分区  1区 (医学)              │
│  学科分类  医学 · 肿瘤学             │
│                                   │
├─────────────────────────────────────┤
│ 投稿信息                            │  ← 新增区块
│ 录用率  ~8%    审稿周期  ~45天      │
│ APC  £9,990    OA期刊              │
├─────────────────────────────────────┤
│ 期刊指标                            │
│ 总发文量  3200    总被引  850000    │  ← 保留但降级显示
│ h指数   450     自引率  1.2%       │
├─────────────────────────────────────┤
│ 年度趋势                            │
│ [图表] 影响因子年度变化趋势           │  ← 新增
│ [图表] 年发文量变化趋势              │
└─────────────────────────────────────┘
```

**关键改动**：
1. 顶部大字号显示 IF + JCR分区 + 中科院分区
2. 新增"投稿信息"区块（录用率、审稿周期、APC）
3. 新增"自引率"指标
4. 去掉"健康评分"
5. 学科主题改为显示大类+小类
6. 新增影响因子年度趋势图

### 5.4 搜索筛选改进

**当前问题**：
- 只有"收录"筛选（DOAJ/MEDLINE/PMC/OA）
- 没有学科分类筛选
- 没有分区筛选

**改进方案**：
1. 筛选面板加"学科分类"行（显示热门大类：医学、生物学、计算机科学等）
2. 筛选面板加"JCR分区"行（Q1/Q2/Q3/Q4 多选）
3. 筛选面板加"中科院分区"行（1区/2区/3区/4区 多选）
4. 排序方式增加"影响因子"（如果有数据的话）

## 七、数据策略详细说明

### 7.1 手动录入方案（第一阶段）

**目标**：覆盖用户搜索量最高的 200 本期刊

**数据来源**：
- JCR 官网（公开数据）
- 知研学术（参考）
- LetPub（参考）
- 期刊官网（录用率/审稿周期）

**入库方式**：
- 创建 `journal_metrics_manual` 表（已有设计）
- 字段：`journal_id`, `impact_factor`, `jcr_quartile`, `cas_partition`, `subject_major`, `subject_minor`, `acceptance_rate`, `review_cycle_days`, `self_citation_rate`, `contributor_id`, `verified`, `created_at`

**审核机制**：
- 手动录入的数据标记为 `verified: true`
- 用户贡献的数据默认为 `verified: false`，需管理员审核

### 7.2 用户贡献方案（第二阶段）

**贡献流程**：
1. 用户进入期刊详情页
2. 点击"补充期刊信息"按钮
3. 填写表单（IF、分区、录用率、审稿周期）
4. 提交后等待审核
5. 审核通过后展示在详情页

**激励措施**：
- 贡献者获得积分
- 积分可兑换 VIP 功能
- 贡献者获得"学术贡献者"徽章

### 7.3 数据展示优先级

当多个数据源都存在时，展示优先级：
1. `journal_metrics_manual`（管理员审核过的）
2. `journal_metrics_contribution`（用户贡献，平均分）
3. OpenAlex 计算的近似指标（标注"估算"）

## 八、总结

**核心结论**：当前产品最大的问题是**缺少用户决策所需的核心数据**（影响因子、分区），导致其他功能价值无法发挥。

**改进路径**：
1. 立即手动录入 Top 200 期刊的 IF/分区数据（1天）
2. 改造列表页和详情页 UI，突出核心指标（2天）
3. 开发用户贡献功能，逐步扩大数据覆盖（2天）
4. 后续考虑爬取或购买数据源

**预期效果**：
- 用户能快速判断期刊水平（IF + 分区）
- 用户能按学科分类筛选期刊
- 用户能查看投稿相关信息（录用率、审稿周期）
- 产品核心价值提升 80% 以上
